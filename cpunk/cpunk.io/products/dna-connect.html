DNA Connect

A fully decentralized messenger with NIST Category 5 post-quantum encryption. No central servers, no phone numbers, no metadata collection. Messages travel through a peer-to-peer DHT network — encrypted end-to-end with Kyber1024 + AES-256-GCM.

Messaging

Encrypted conversations, zero compromise

Every message is end-to-end encrypted and transported through a distributed hash table. There are no central servers to compromise, subpoena, or shut down.

1:1 Direct Messages

Private conversations encrypted with Kyber1024 + AES-256-GCM. Address contacts by their DNA name — no phone numbers or email needed.

  • Delivery and read receipts
  • Cross-device sync via seed phrase
  • Background push notifications (Android)

Group Messaging

DHT-based decentralized groups with GEK encryption — 200x faster than per-recipient encryption. Automatic key rotation on member changes.

  • Group ownership transfer
  • P2P group invitations
  • GEK versioning for forward security

Offline Message Queue

Messages are stored in distributed mailboxes on the DHT with a 7-day TTL. No always-on requirement — messages wait until the recipient comes online.

  • Spillway Protocol v2 for efficient retrieval
  • Daily bucket message organization
  • "Never Give Up" retry system
Social

Public feeds & user profiles

Public Feeds (DNA Board)

Topic-based public discussion feeds stored entirely on the DHT. Create, subscribe, comment, and vote — all cryptographically signed with Dilithium5 for author verification.

  • Categories: General, Technology, Help, Announcements, Trading
  • Comments: Threaded replies with newest-first ordering
  • Voting: Community-driven content curation
  • Community feeds: Bug Reports, Feature Requests, Official Announcements

Security

NIST Category 5 — maximum quantum resistance

256-bit security level. Protected against both current classical attacks and future quantum computers. Your keys never leave your device.

Algorithm Standard Purpose
Kyber1024 ML-KEM-1024 (FIPS 203) Key encapsulation
Dilithium5 ML-DSA-87 (FIPS 204) Digital signatures
AES-256-GCM NIST Symmetric encryption
SHA3-512 NIST Hashing
BIP39 Bitcoin Standard 24-word seed phrase recovery
No Central Servers
Messages travel peer-to-peer through the DHT. Nothing to compromise or subpoena.
No IP Leakage
Your real IP is never exposed to contacts or third parties. DHT-only transport.
No Metadata

No Phone Number
Identity via DNA name tied to your cryptographic key pair. No personal data required.
Wallet

Multi-chain wallet, built into the messenger

A single BIP-39 seed phrase derives wallets across four blockchains. Send and receive tokens without leaving the conversation — no address copying, no app switching.

Multi-Chain Architecture

One wallet interface, four blockchains managed from a single unified view. Each chain uses its native signing algorithm.

In-Chat Transfers

Send tokens to any DNA contact directly inside a conversation. The recipient's wallet address is resolved automatically from their DNA identity.

DEX Integration

Swap tokens directly in the app via Uniswap, PancakeSwap, Jupiter, and Cellframe DEX. Best-rate routing across multiple pools.

Cellframe

Post-quantum blockchain with Dilithium signatures. Native CPUNK & CELL tokens.

Ethereum

Full mainnet support with ERC-20 tokens, dynamic gas estimation, and DEX swaps.

Solana

High-throughput chain with Ed25519 signatures, SPL token transfers, and Jupiter swaps.

TRON

TRON mainnet with TRC-20 token support. secp256k1 signing with Base58Check addresses.

Architecture

Serverless peer-to-peer network

DNA Connect runs on Nodus — a post-quantum Kademlia DHT where RSA-2048 is replaced with Dilithium5 post-quantum signatures. All DHT values are mandatory-signed.

DNA Connect App

Cross-platform UI built with DNA Connect. Runs natively on Android, Linux, and Windows. iOS and Web planned.

C Library (libdna_engine)

Core cryptography, DHT operations, and local database in a native C library. Handles all encryption, signing, and network transport.

DNA Nodus (Bootstrap)

Lightweight DHT bootstrap nodes for peer discovery. No message relay — they only help peers find each other. No logging, no content access.
