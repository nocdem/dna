# Archived Docs — `/opt/dna/docs/`

Files in this directory are **historical**. They describe earlier designs, obsolete plans, or superseded analysis.

- **Not authoritative.** Do not treat archived content as current behavior or policy.
- **Kept for history.** Provides context for past decisions and parameter choices.
- **Each file starts with an archive header** noting the date and reason.

Current authoritative docs live one level up in `/opt/dna/docs/`. For per-project docs see `messenger/docs/`, `nodus/docs/`, `dnac/docs/`.
